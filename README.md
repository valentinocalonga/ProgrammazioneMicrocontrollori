- UART serial command interface  
- GPIO hardware abstraction  
- LED control driver  
- Switch input handling  
- Timer-based delay generation  
- Modular embedded firmware architecture  

Developed for the **Microcontroller Programming course (SUPSI)**.

---

## ✨ Overview

The firmware allows controlling on-board LEDs through:

- serial commands received via **UART4**
- direct interaction with **hardware switches**
- autonomous sequences based on **timer delays**

The project demonstrates:

- direct register-level peripheral configuration  
- polling-based embedded programming  
- modular firmware driver design  
- simple command parsing  
- hardware abstraction techniques  
- timer usage in **16-bit and 32-bit modes**

## 🔹 GPIO Driver

Responsible for low-level hardware configuration.

Features:

- LED port configuration  
- switch input configuration  
- direct register manipulation  

Key API:

```c
void initLeds(void);
void initSwitches(void);
int readSwitch(int sw);
void togglePort(char port, int bit);
```

---

## 🔹 LED Abstraction Layer

Provides logical LED control independent from register details.

```c
void setLed(int ledNumber, int state);
void toggleLed(int ledNumber);
```

---

## 🔹 UART Driver

Polling-based UART4 communication driver.

Features:

- character transmission  
- string transmission  
- blocking string reception  

```c
void initUART(unsigned int baudrate);
void putU4(int c);
void putU4_string(const char *str);
char getU4(void);
char *getString(void);
```

---

## 🔹 Timer Modules

### Generic Timer Module (16-bit)

Used for simple delay demonstrations and exercises.

```c
void initTimer2(void);
void delay(int value);
```

### Timer2/Timer3 32-bit Exercise Module

Implements LED sequencing with **timer polling only (no interrupts)**.

Features:

- Timer2 + Timer3 combined in **32-bit mode**
- delay generated by monitoring **TMR2/TMR3 registers**
- sequential LED activation / deactivation via UART commands  

Example API:

```c
void initTimer23_32bit(void);
void waitDelay(void);
void TIMER_Serie4_processCommand(char *cmd);
void TIMER_Serie4_Es1_NOINTERRUPT(void);
```

## 💬 Example Serial Commands

Using a serial terminal (PuTTY, SerialTools, TeraTerm):

```
ledon
ledoff
```

Behaviour:

- `ledon` → LEDs turn on sequentially (LED0 → LED7)  
- `ledoff` → LEDs turn off sequentially (LED7 → LED0)  
- delay between LEDs is generated by **Timer2/3 in 32-bit polling mode**
